# 零一优创·项目背景与上下文

## 一、公司背景

**零一优创 (One Up)** 是一家科技创新教育公司，专注于辅导1-9年级学生完成科技创新项目，参加全国青少年科技创新大赛、北京金鹏科技论坛等竞赛。

### 业务模式
- **B2B**：与学校合作，提供20课时的科创项目课程
- **B2C**：面向个人家庭，提供24-30课时的一对一/小组项目辅导
- **创始人**：Wei，具备Arduino、ESP32、IoT、3D打印、机器学习等技术专长

### 教育理念
- **"以科创为体，以AI为手"** — 科技创新是核心，AI是辅助工具
- **学生是创造者，不是观察者** — 强调学生自己动手、自己思考，拒绝套件式教学
- **真实问题驱动** — 每个项目从学生真实生活中的问题出发，不是预设题目

## 二、11步教学方法论

这是零一优创的核心方法论，经过多年实践验证。平台的引导流程直接基于此方法论设计。

### 完整流程

1. **科技启发与问题发现** — 通过案例激发学生兴趣，引导学生从生活中发现真实问题
2. **需求调研与验证** — 设计问卷、进行用户访谈，验证问题的真实性和普遍性
3. **本质分析与方案设计** — 分析问题根因，设计解决方案，进行技术选型
4. **方案可视化** — 画系统架构图、流程图，让方案从抽象变具体
5. **原型制作** — 动手搭建硬件、编写代码，完成第一个可运行版本（通常占4节课）
6. **测试与验证** — 设计测试方案，收集数据，验证方案是否有效
7. **迭代与优化** — 基于测试结果改进方案，可能返回第5步重做
8. **成果展示** — 制作PPT、展板、录制视频，准备竞赛答辩

### 关键教学原则
- 每一步都有明确的**产出物**（文档、原型、数据、报告）
- 老师的角色是**引导者**，通过提问帮助学生思考，而不是直接给答案
- 允许失败和迭代，**过程比结果重要**
- 所有文档必须保持**学生自己的语言**，不能成人化

## 三、真实教学场景参考

以下是典型的师生互动片段，反映了教学引导的风格和分寸。这些是平台AI引导设计的重要参考。

### 场景1：引导四年级学生发现问题

```
老师：最近有没有什么事情让你觉得不方便的？
学生：嗯……没有……
老师：想想看，比如早上起床到上学这段时间，有没有什么烦恼？
学生：早上老是找不到红领巾！
老师：哦？那你通常怎么办？
学生：就翻书包、翻柜子，有时候找了五分钟。
老师：那你们班其他同学也会这样吗？
学生：好多人都会！有时候老师还会批评没带红领巾的同学。
老师：那你觉得，如果有什么东西能帮你解决这个问题就好了？
学生：要是有个东西能告诉我红领巾在哪就好了！
老师：这是个很好的想法。那我们想想，怎么才能知道红领巾在哪呢？
```

**要点：** 老师没有直接说"你可以做个智能找红领巾的设备"，而是通过追问一步步引导学生自己到达这个结论。

### 场景2：引导初中生优化技术方案

```
老师：你的传感器数据波动很大，你觉得可能是什么原因？
学生：是不是传感器坏了？
老师：有可能。但在换传感器之前，我们可以先做什么来判断？
学生：……多测几次？
老师：对！我们可以采集多组数据看看。另外，你觉得环境因素有没有影响？
学生：哦，可能是温度？今天挺热的。
老师：很好的观察。那你打算怎么验证这个猜想？
学生：我可以在不同温度下测试？
老师：没错。记得把数据记录下来，用表格对比。
```

**要点：** 学生说"传感器坏了"是最直觉的反应，老师没有否定，而是引导他用科学方法排查。

### 场景3：引导学生在多个选题间做选择

```
老师：你列了三个问题——奶奶经常忘记吃药、小区流浪猫没人喂、教室黑板反光。你觉得哪个你最想解决？
学生：我最想帮奶奶。
老师：为什么呢？
学生：因为奶奶年纪大了，有时候忘记吃药很危险。
老师：这个出发点很好。那你觉得这个问题用技术能解决吗？
学生：可以做个提醒器？
老师：嗯，市面上有没有类似的产品？
学生：有啊，手机闹钟就行。
老师：那为什么奶奶不用手机闹钟？
学生：她不太会用手机……而且有时候听不见。
老师：所以你的解决方案需要解决什么问题？
学生：要简单，声音要大，最好还能让家人知道她有没有吃药。
```

**要点：** 通过追问"为什么不用现有方案"，帮助学生发现真正的需求差距——这是创新的起点。

## 五、典型学生项目案例

以下是零一优创辅导过的真实项目类型，供AI理解"学生科创项目"的范围：

## 六、设计偏好

### 整体风格
- **温暖、友好** — 面向中小学生和家长，不要冰冷的科技感
- **避免典型的"科技蓝"** — 偏向温暖色系（橙、绿、黄等）
- **简洁不简陋** — 保持专业感但不要企业级的严肃

### 品牌色参考
- 主色：暖橙色系（如 #FF8C42 或类似）
- 辅助色：温暖的绿色、米黄色
- 强调色：根据具体模块调整
- 文字色：深灰而非纯黑（如 #333333）

### 字体
- 中文：思源黑体（Noto Sans CJK SC）或系统默认
- 英文/代码：Inter 或 Mono字体
- 注意中文排版需要更大的行间距

### 交互特点
- 学生可能用手机/iPad访问，触控友好
- 学生打字速度慢，减少长文本输入，多用选择和结构化表单
- AI对话气泡的设计要有温度，不像客服机器人
- 进度可视化要有成就感（完成一步有明确的视觉反馈）

## 七、竞赛背景知识

### 主要目标竞赛
1. **全国青少年科技创新大赛** — 最权威的青少年科创竞赛
2. **北京金鹏科技论坛** — 北京市级重要竞赛
3. **其他**：宋庆龄少年儿童发明奖、明天小小科学家等

### 竞赛评审关注点
- **真实性** — 项目是否由学生本人完成（最重要！）
- **创新性** — 是否有独特的思路和解决方案
- **科学性** — 是否有科学的研究方法和数据支撑
- **实用性** — 是否能解决实际问题
- **完整性** — 从问题发现到成果展示的完整过程

### 竞赛材料包（平台最终需要帮学生产出的全套材料）
- 研究论文/研究报告（.md → .docx）
- 研究方案/开题报告
- 技术方案
- 研究日志
- PPT演示文稿
- 项目介绍视频（平台不负责，学生自己录制）
- 展板（展示用海报）
- 照片记录（贯穿全程）

## 八、部署与技术环境

### 目标用户设备
- 主要：家长手机（iOS/Android）和iPad
- 次要：电脑浏览器
- **移动端优先设计**

### 国内部署考虑
- 用户全在中国大陆
- 需要考虑网络访问速度
- Vercel在国内可能有延迟
- Supabase服务器在海外可能有延迟
- 可能需要在后续迭代中迁移到国内云服务（腾讯云/阿里云）
- **MVP阶段先用Vercel + Supabase快速上线验证，后续再考虑迁移**

### API相关
- 核心AI引擎使用Claude API（Anthropic）
- 需要处理国内API调用的网络问题（可能需要代理或使用国内接入点）
