# Writing Guidelines

## Universal Rules (All Grades)

### Forbidden Patterns

**NEVER use metaphors for technical components:**
- ❌ "ESP32就像大脑一样控制着整个系统"
- ❌ "摄像头是装置的眼睛"
- ❌ "传感器充当了电子鼻子的角色"
- ❌ "灯带就像一条彩虹"
- ✅ "ESP32主控芯片负责数据采集和处理"
- ✅ "摄像头模块用于图像识别"
- ✅ "MQ-2烟雾传感器检测空气中的烟雾浓度"
- ✅ "WS2812B灯带显示不同颜色表示系统状态"

**NEVER use vague effect descriptions in abstracts:**
- ❌ "取得了良好的效果"
- ❌ "性能得到了显著提升"
- ❌ "用户反馈很好"
- ✅ "测距精度达到±10cm"
- ✅ "检测准确率为97.4%"
- ✅ "95%的测试用户表示操作便捷"

**NEVER exaggerate or fabricate data.**

**NEVER use overly sophisticated language that doesn't match the student's grade level.**

### Required Sections

Every paper, regardless of grade, must include:

1. **个人贡献** — positioned after abstract, before background
2. **心得体会** — near the end of the paper
3. **致谢** — final section before references

### Abstract Writing Formula

Regardless of grade, every abstract follows this pattern (with age-appropriate language):

```
[问题背景 1-2句] + [技术方案 1-2句] + [量化结果 2-3句] + [创新亮点 1-2句] + [应用价值 1句]
```

The abstract should give an evaluator a complete picture of the project in under 30 seconds of reading.

## 小学低年级 Writing Style

### Language
- 完全口语化，第一人称"我"
- 可以出现"爸爸""妈妈""老师"等称呼
- 可以表达情感："我很开心""我有点担心"
- 句子短，结构简单
- 可以用"首先...然后...最后..."等连接词

### Tone
- 像在讲一个自己做实验的故事
- 充满好奇心和探索精神
- 遇到困难时诚实表达
- 成功时的真实喜悦

### Example Expressions
- "有一天，我发现..."
- "我想了很久，觉得可以..."
- "我试了好几次，第一次失败了，因为..."
- "后来爸爸建议我..."
- "最后终于成功了！"
- "通过这个项目，我学到了..."

### Avoid
- 任何学术化表达
- 复杂的技术术语（用简单描述代替）
- 长句子和复杂句式
- 被动语态

---

## 小学高年级 Writing Style

### Language
- 自然的学生表达，允许适度口语化
- 第一人称"我"为主
- 可以提到家人和老师的帮助
- 技术术语出现时给简单解释
- 有一定的条理性，但不要过于正式

### Tone
- 有条理地讲述发明过程
- 体现自主思考和解决问题
- 数据意识（会用表格和数字说明问题）
- 适当的反思和总结能力

### Example Expressions
- "在日常生活中，我经常遇到这样的问题..."
- "为了了解大家是否也有同样的困扰，我设计了一份问卷..."
- "经过对比分析，我最终选择了方案二，因为..."
- "在制作过程中，遇到了一个困难：...我尝试了...最终..."
- "测试结果显示..."
- "我认为还可以从以下方面改进..."

### Data Presentation
- 问卷结果用百分比和简单图表
- 测试数据用表格
- 对比用表格（方案A vs 方案B）
- 材料清单用表格

### Avoid
- 过于学术化的表达
- "本系统""本装置"等指代（用"我的作品""这个装置"代替）
- 复杂的统计分析
- 文献引用（可以简单提到"我查阅了资料发现..."）

## 初中 Writing Style

### Language
- 严谨正式，但不必完全学术化
- "我"和"本系统/本装置/本项目"混用
  - 描述个人行为时用"我"："我设计了..."
  - 描述系统特性时用"本系统"："本系统采用..."
- 技术术语直接使用，必要时在首次出现时给出简要解释
- 句式完整，逻辑连接词恰当

### Tone
- 工程师视角，注重逻辑和数据
- 客观描述技术方案和测试结果
- 遇到问题时用分析的方式说明原因和解决方法
- 创新点用对比数据支撑而非主观评价

### Example Expressions
- "通过调查研究发现，目前...存在以下问题..."
- "针对上述问题，本项目设计了一套..."
- "系统采用ESP32-S3作为主控芯片，通过SPI总线与DW1000 UWB模块通信"
- "在调试过程中，发现传感器读数存在±5%的波动，经分析是由于...，通过增加中值滤波算法解决了该问题"
- "测试结果表明，本系统在...指标上达到了设计要求"
- "与传统方案相比，本项目在成本上降低了60%，同时精度提升了40%"

### Data Presentation
- 所有技术指标量化
- 多轮测试数据表格
- 对比分析表格（本项目 vs 现有方案）
- 硬件连接表、元器件参数表
- 用户测试统计

### Avoid
- 口语化表达："我觉得""我试了试"
- 模糊描述："效果很好""速度很快"
- 过度学术化：不需要文献综述
- 提到家长帮助（在个人贡献中提"在老师指导下"即可）

---

## 高中 Writing Style

### Language
- 学术化，"本研究""本文""本系统"
- 被动语态和主动语态结合
- 专业术语规范使用
- 引用格式严格（GB/T 7714）

### Tone
- 研究者视角
- 注重理论依据和方法论
- 批判性思维（讨论局限性和改进空间）
- 文献驱动（现有研究 → 不足 → 本研究的切入点）

### Example Expressions
- "据统计数据显示，我国每年因...造成的损失达到..."
- "文献[3]提出了一种基于...的方法，但存在...的局限"
- "本研究提出一种基于...的解决方案"
- "采用TWR（Two-Way Ranging）双向测距算法，通过测量信号往返飞行时间计算距离"
- "实验结果表明，在95%置信区间内，系统测距误差为±10cm"
- "本研究的主要贡献包括：（1）...（2）...（3）..."

### Data Presentation
- 正式的实验方案（测试环境、条件、方法）
- 对照实验设计
- 统计分析（均值、标准差、置信区间）
- 误差分析
- 多维度对比表格

### Avoid
- 任何口语化表达
- 第一人称"我"（用"本文作者""本研究"替代）
- 情感化表达
- 非学术来源的引用

## 个人贡献 Writing Guide

### Purpose
让评委快速了解学生在项目中的真实参与程度，建立项目真实性和学生能力的信任。

### 小学低
```
在这个项目中，我自己完成了以下工作：
- 发现了[问题]
- 自己设计了[方案的某部分]
- 动手制作了[具体内容]
- 测试了[具体内容]
在[具体环节]，爸爸/妈妈帮助了我[具体帮助内容]。
老师指导我[具体指导内容]。
```

### 小学高
```
在本项目中，我主要完成了以下工作：
1. 独立完成了问题调研，设计并发放了XX份问卷
2. 提出了[核心设计思路]
3. 独立完成了[硬件组装/编程/3D设计]等工作
4. 设计测试方案并完成了全部测试
5. 撰写了项目报告
在[具体环节]得到了指导老师的帮助。
```

### 初中
```
本项目中，作者主要负责以下工作：
1. 问题调研与需求分析
2. 系统方案设计与技术选型
3. [具体硬件]的组装与调试
4. [具体软件功能]的程序编写
5. 系统测试方案的设计与执行
6. 全部项目文档的撰写
在[UWB通信协议调试]等环节得到了指导老师的技术指导。
```

### 高中
```
本研究中，作者的主要贡献如下：
1. 完成了相关领域的文献调研与分析
2. 提出了基于[技术]的[方案名称]
3. 独立完成了系统的硬件设计与软件开发
4. 设计并执行了完整的实验验证方案
5. 完成了数据分析与论文撰写
[具体环节]在指导教师的建议下进行了优化改进。
```

## 心得体会 Writing Guide

### Purpose
展示学生通过项目获得的素养和能力提升，这是评委评价教育价值的重要依据。

### 应涵盖的维度
1. **知识技能**：学到了什么具体技术或知识
2. **思维能力**：解决问题的方法、系统思维、批判性思考
3. **实践体验**：动手能力、调试耐心、失败中学习
4. **价值认识**：对科技创新的理解、对社会问题的关注
5. **未来启发**：对未来学习或项目的期待

### 小学低
侧重感受和简单的知识收获，用"我学到了..."的句式。

### 小学高
增加对过程的反思，可以讨论失败的经历和从中学到的东西。

### 初中
增加对工程思维和方法论的体会，可以讨论技术决策的过程。

### 高中
增加学术层面的反思，讨论研究方法的心得和学术态度的培养。

## 致谢 Writing Guide

### 致谢对象（按惯例顺序）
1. 指导老师（必须）
2. 父母/家人（必须）
3. 学校/实验室（提供场地、设备）
4. 帮助测试的同学/志愿者
5. 提供技术帮助的其他人

### 注意
- 真实、具体（"感谢张老师在电路调试方面的耐心指导"比"感谢老师的帮助"好）
- 篇幅适当，不过长
- 语气真诚

## Common Pitfalls to Avoid

### ❌ 摘要过于笼统
```
本项目设计了一个智能装置，效果良好，具有一定的创新性和实用价值。
```
### ✅ 摘要具体有力
```
本项目设计了一套基于UWB测距的便携式视力测量仪，利用DW1000超宽带模块实现±10cm精度的5米距离自动校准，采用4.2寸电子墨水屏显示E字视标（对比度≥85%）。经测试，系统测距成功率达96%，单次测试时间约3分钟，续航超过8小时。主要创新点包括UWB精确测距替代传统卷尺、电子墨水屏高对比度视标显示。
```

### ❌ 创新点空泛
```
本项目具有很强的创新性，设计独特，效果好。
```
### ✅ 创新点有数据支撑
```
创新点1：UWB精确测距替代传统卷尺
- 传统方案：人工卷尺测量，误差±20cm
- 本项目方案：UWB自动测距，精度±10cm
- 提升：测距精度提升50%，且无需人工操作
```

### ❌ 测试结果模糊
```
测试了很多次，都成功了，效果很好。
```
### ✅ 测试结果量化
```
进行了20次独立测试，其中19次成功完成全流程，成功率95%。失败的1次是由于UWB信号被金属物体遮挡导致，重新调整位置后恢复正常。
```

### ❌ 跳过失败经历
```
制作过程很顺利，一切按计划完成。
```
### ✅ 记录真实的迭代过程
```
在调试UWB测距模块时，最初的读数波动很大（±30cm）。经分析发现是因为天线方向不对，两端的天线没有对准。调整天线角度并增加中值滤波算法后，测距精度稳定在±10cm以内。
```
