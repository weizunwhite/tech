# 智能药盒项目 B端教案 - 第1节课

**课程信息**
- **项目名称**：智能药盒提醒系统
- **授课模式**：一对一
- **课程性质**：科技创新竞赛项目
- **课时安排**：2小时（120分钟）
- **适用年级**：初中生（7-9年级）

**本节对应教学步骤**：
- 步骤1：发现问题与共情
- 步骤2：定义课题与成功标准

---

## 一、教学目标

### 知识目标
- 了解老龄化社会背景下的用药管理问题
- 认识智能硬件在养老领域的应用价值
- 初步了解物联网技术的基本概念

### 能力目标
- 学会从生活中发现真实问题
- 掌握用户访谈与需求分析方法
- 能够将模糊想法转化为清晰的项目目标
- 学会制定可量化的成功标准

### 素养目标
- 培养对老年群体的同理心与社会责任感
- 建立"技术为人"的价值观
- 形成项目化思维的基本意识

---

## 二、教学准备

### 教师准备
**课前材料**：
- [ ] 老年人用药场景的视频/图片（2-3个案例）
- [ ] 市面上已有的药盒产品图片（传统分格药盒、定时药盒、智能药盒）
- [ ] 项目最终成品效果图/演示视频（用于激发兴趣）
- [ ] 用户访谈记录表（空白模板，打印1份）
- [ ] 需求清单表格（空白模板）
- [ ] 项目成功标准表格（空白模板）

**教学工具**：
- 白板/白纸（用于画思维导图）
- 便利贴（不同颜色，用于分类整理需求）
- 计时器

**预习任务**（提前1天发给学生）：
```
【课前任务】
请你访谈一位老年人（可以是爷爷奶奶、外公外婆、或邻居长辈），了解：
1. 他/她每天需要吃什么药？
2. 每种药什么时候吃？一次吃几颗？
3. 他/她在吃药时遇到过什么困难或麻烦？
4. 有没有忘记吃药或吃错药的经历？

请简单记录下来，上课时分享。
```

## 三、教学过程

### 环节一：课前暖场 + 作业分享（15分钟）

**时间**：0:00-0:15

#### 1.1 破冰聊天（5分钟）

**目的**：建立师生关系，了解学生背景

**话术参考**：
```
老师："你好！很高兴今天能和你一起做这个项目。先介绍一下，
我是[老师名字]，我们接下来10节课要一起完成一个很有意义的作品。
在开始之前，我想先了解一下你——

1. 你平时对什么感兴趣？（编程、硬件、3D打印...）
2. 之前做过类似的项目吗？
3. 你家里有老人需要经常吃药吗？

轻松聊就好，没有标准答案。"
```

**教师记录要点**：
- 学生技术基础（编程/硬件经验）
- 学生性格特点（主动/被动）
- 家庭背景（是否有老人）

#### 1.2 课前访谈作业分享（10分钟）

**引导提问**：
```
老师："很好！那我们先来看看你课前的访谈作业。你访谈了谁？
能跟我讲讲你的发现吗？"

（学生分享）

追问：
- "他/她吃药的时候，你观察到最大的困难是什么？"
- "有没有让你印象深刻的细节？"
- "你觉得为什么会出现这个问题？"
```

**教师引导要点**：
- 肯定学生的观察
- 引导学生说出具体场景（不要泛泛而谈）
- 记录下关键痛点（板书或记在纸上）

**预期输出**：
```
学生访谈发现（示例）：
✓ 奶奶有时忘记吃药，要家人提醒
✓ 药盒上的字太小，看不清楚
✓ 有一次吃重复了，因为忘了早上已经吃过
✓ 爸妈不在家时，没人提醒
```

---

### 环节二：问题发现与场景共情（35分钟）

**时间**：0:15-0:50

#### 2.1 视频案例观看 + 讨论（15分钟）

**播放材料**：
- 视频1：老年人因忘记吃药导致病情加重（2分钟）
- 视频2：老年人药盒凌乱，分不清哪个是哪个（1分钟）

**观看后引导讨论**：
```
老师："看完这两个视频，结合你的访谈，我们来梳理一下——
老年人在吃药这件事上，到底遇到了哪些问题？"

（学生回答，老师板书）

追问：
- "为什么会忘记？"（记忆力衰退）
- "为什么会搞混？"（药物种类多、视力下降）
- "除了忘记和搞混，还有什么问题？"（没人提醒、不方便）
```

**教师活动**：
在白板/白纸上绘制**问题清单思维导图**：

```
        老年人用药问题
         /    |    \
      记忆   视力   环境
       |      |      |
    忘记吃  看不清  无人提醒
    吃重复  拿错药  不方便
```

#### 2.2 用户画像与场景还原（20分钟）

**活动：构建用户画像**

```
老师："我们来具体想象一个人——就叫'李奶奶'，75岁，
她每天需要吃3种药，我们来设计她的一天。"
```

**引导学生填写用户画像表**：

| 项目 | 内容 |
|------|------|
| 姓名 | 李奶奶 |
| 年龄 | 75岁 |
| 健康状况 | 高血压、糖尿病、轻度健忘 |
| 药物清单 | 降压药（早）、降糖药（早晚）、护肝药（晚） |
| 生活状态 | 独居，子女工作日不在家 |
| 使用能力 | 会用简单的电器，但不会智能手机 |

**情景演绎（重点）**：

```
老师："现在，你来扮演李奶奶。早上8点到了，闹钟响了，
你该吃降压药了。但是——

场景1：你忘记了今天早上是否已经吃过了，怎么办？
场景2：桌上有3个药瓶，你视力不好，看不清哪个是降压药
场景3：你在卧室看电视，药盒在客厅，没有提醒，你忘了吃药

（让学生说出感受）

现在你能理解李奶奶的困扰了吗？她最需要什么？"
```

**预期学生回答**：
- 需要明确的提醒（到时间自动提醒）
- 需要清晰的分类（不会拿错）
- 需要简单的确认方式（知道是否吃过）
- 提醒方式要老人能注意到（不能只是静音通知）

**教师记录**：
把学生说出的需求用**便利贴**写下来，贴在白板上，后面会用到。

---

### 环节三：现有方案调研与分析（20分钟）

**时间**：0:50-1:10

#### 3.1 现有产品调研（10分钟）

**展示图片/实物**：
1. 传统7格药盒（按周一到周日分格）
2. 定时提醒药盒（带蜂鸣器）
3. 智能药盒（联网款，价格500-2000元）

**引导分析**：
```
老师："市面上已经有一些解决方案，我们来看看它们的优缺点。"

（逐个分析）

传统7格药盒：
老师："这种药盒好在哪里？不足在哪里？"
学生回答，老师补充：
✓ 优点：便宜、简单
✗ 缺点：没有提醒、需要每周手动分药、容易搞混

定时提醒药盒：
✓ 优点：有提醒功能
✗ 缺点：只有蜂鸣声，老人可能听不见；还是要手动分药

智能药盒：
✓ 优点：功能强大，可以联网、远程监控
✗ 缺点：价格贵（500-2000元）、操作复杂、老人学不会
```

**绘制对比表**（师生共同完成）：

| 产品类型 | 优点 | 缺点 | 价格 |
|---------|------|------|------|
| 传统分格药盒 | 简单便宜 | 无提醒、需手动分药 | 10元 |
| 定时药盒 | 有蜂鸣提醒 | 提醒单一、仍需分药 | 50元 |
| 智能药盒 | 功能丰富、可联网 | 太贵、太复杂 | 500-2000元 |
| **我们的方案** | ？？？ | ？？？ | ？？？ |

#### 3.2 我们的机会点（10分钟）

**引导讨论**：
```
老师："看完这些产品，你觉得我们的机会在哪里？
我们能做一个什么样的药盒，既解决老人的问题，
又比现有产品更好？"

关键引导问题：
1. "提醒方式能不能更多样？"（视觉+听觉+触觉）
2. "操作能不能更简单？"（一键确认）
3. "成本能不能更低？"（3D打印+开源硬件）
4. "分类能不能更科学？"（按服药频率和剂量分类）
```

**教师总结**：
```
我们的创新方向（写在白板上）：
✓ 多方式提醒（灯光+语音+手环震动）
✓ 智能分类（按一天几次 × 一次几颗）
✓ 简单确认（双击按钮）
✓ 低成本（3D打印，约200元）
✓ 可记录（屏幕显示已吃次数）
```

---

### 环节四：定义课题与成功标准（30分钟）

**时间**：1:10-1:40

#### 4.1 明确项目课题（10分钟）

**引导确定项目名称**：
```
老师："好，现在我们要给这个项目起个名字。
你觉得叫什么好？"

（学生提议）

老师引导：
- 要体现"智能"
- 要体现"提醒"功能
- 要让评委一听就知道是什么

最终确定：智能药盒提醒系统
（或学生提出更好的名称）
```

**撰写项目一句话描述**：
```
和学生一起完成这句话（写在记录本上）：

"这是一个【        】，它能帮助【        】解决【        】问题，
通过【        】技术实现【        】功能。"

填空示例：
"这是一个智能药盒提醒系统，它能帮助老年人解决忘记吃药、
吃错药的问题，通过物联网和智能硬件技术实现多方式提醒、
自动分类和服药记录功能。"
```

#### 4.2 定义成功标准（20分钟）

**引导制定标准**：
```
老师："我们要做这个项目，怎么算成功？怎么算失败？
我们需要定一些具体的标准。"

提示学生从这几个方面思考：
1. 功能：必须实现哪些功能？
2. 性能：要达到什么指标？
3. 易用性：老人能不能用？
4. 成本：不能超过多少钱？
5. 时间：什么时候要完成？
```

**师生共同填写"项目成功标准表"**：

```
【项目成功标准表】

一、必须实现的核心功能（MVP）：
✓ 1. 定时提醒功能（准时率>95%）
✓ 2. LED灯光提醒（能准确点亮对应抽屉）
✓ 3. 语音播放提醒（音量足够大，老人能听见）
✓ 4. 手环震动提醒（延迟<3秒）
✓ 5. 按钮确认功能（双击成功率>90%）
✓ 6. 屏幕显示服药次数

二、性能指标：
- 时间准确性：误差 ±5秒以内
- 通信延迟：<3秒
- 系统稳定性：连续运行24小时不崩溃
- 按钮响应率：>90%

三、易用性标准：
- 老人能独立完成"双击确认"操作
- 不需要看说明书就知道哪个抽屉该拿药
- 设置时间的操作不超过5步

四、成本限制：
- 总成本 <300元
- 单次材料成本 <250元

五、时间节点：
- 第7节课：完成原型制作
- 第10节课：完成所有测试和优化
- 比赛提交前：完成项目报告

六、加分项（如果时间允许）：
- 家属远程监控（可选）
- 语音播报药品名称（可选）
- 更换电池提醒（可选）
```

**重要**：让学生亲笔抄写一遍成功标准，签字确认。
这是后续遇到困难时的"锚点"——提醒学生我们的目标是什么。

---

### 环节五：课堂总结与下节预告（15分钟）

**时间**：1:40-1:55

#### 5.1 知识点回顾（5分钟）

**提问检查**：
```
老师："今天这节课，我们做了很多事情，我来考考你，
你还记得吗？"

问题：
1. 老年人吃药遇到的最大问题是什么？（记忆、视力、无人提醒）
2. 我们的项目和市面上的智能药盒相比，优势在哪？
   （多方式提醒、低成本、简单易用）
3. 我们定的核心功能有哪些？（6个核心功能）
4. 我们的成功标准是什么？（让学生说出2-3个关键指标）
```

#### 5.2 成果物检查（5分钟）

**本节课学生应该拥有的成果**：
- [ ] 用户画像表（李奶奶的基本信息）
- [ ] 问题清单（老年人用药痛点3-5条）
- [ ] 现有方案对比表
- [ ] 项目一句话描述
- [ ] 项目成功标准表（手写签字版）

**教师检查**：确认学生的记录本上有以上内容，拍照留档。

#### 5.3 课后任务布置（5分钟）

**任务1：资料收集**
```
【作业1】上网搜索以下内容，每个找2-3个案例/图片：
1. ESP32开发板长什么样？能做什么？
2. WS2812灯带是什么？有什么效果？
3. 什么是物联网？什么是MQTT？
4. 类似的智能药盒产品有哪些？

要求：截图或打印，带到下节课
```

**任务2：思维准备**
```
【作业2】思考题（下节课讨论）：
1. 如果要实现"定时提醒"功能，需要哪些硬件？
2. 如果要实现"手环震动"，药柜和手环怎么通信？
3. 你觉得我们的方案有哪些技术难点？

不需要写答案，思考就好，下节课我们一起解决。
```

**任务3：继续访谈（可选）**
```
【作业3】如果有机会，再访谈1-2位老人，问问：
"如果有一个能自动提醒吃药的智能药盒，你希望它有什么功能？"

记录下他们的想法，可能会有新的灵感。
```

## 备注

**教学要点**：
1. 第一节课最重要的是**建立信任**和**激发兴趣**
2. 要让学生真正理解"为什么做这个项目"，而不是老师指定的任务
3. 成功标准的制定要让学生参与，这样后续才有ownership
4. 一对一教学的优势是可以根据学生反应灵活调整，不要死板按时间

**常见问题处理**：
- 如果学生没做课前访谈：现场打电话给家长，让家长描述老人吃药情况
- 如果学生对老年人没有共情：多用视频案例、角色扮演来引导
- 如果学生觉得项目太难：强调"我们一步一步来，每节课只学一小块"
- 如果学生想法太多：先肯定，再引导聚焦到核心功能（MVP思维）
